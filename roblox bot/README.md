# Playerok Roblox Robux bot + Telegram-панель

Два независимых процесса, общающихся через общую SQLite БД (`bot.db`):

- **`playerok_bot.py`** - слушает Playerok, обрабатывает сделки категории
  Roblox, создаёт заказы на 2faroblox.com (Swizzyer API).
- **`tg_admin_bot.py`** - Telegram-бот с панелью: настройка текстов (вкл/выкл
  + редактирование, как на скрине "Тексты для заказов Roblox"), просмотр
  логов/ошибок, управление каталогом товаров.

## Установка

```bash
pip install -r requirements.txt
```

## Настройка (`config.py`)

1. **PLAYEROK_TOKEN** / **PLAYEROK_USER_AGENT** - токен авторизованной сессии
   Playerok.
2. **SWIZZYER_API_KEY** - ключ `swz_live_...` из дашборда 2faroblox.com
   (scope `read_write`).
3. **ROBLOX_GAME_CATEGORY_NAME** - по этой строке бот определяет, что сделка
   относится к Roblox (сверяется с названием игры/категории предмета).
4. **TG_BOT_TOKEN** - токен Telegram-бота админ-панели (от @BotFather).
5. **TG_ADMIN_IDS** - список Telegram user_id, которым разрешён доступ к
   панели.

## Запуск

```bash
# терминал 1
python playerok_bot.py

# терминал 2
python tg_admin_bot.py
```

## Telegram-панель

### \U0001F47E Тексты для заказов Roblox

Список текстов с переключателями вкл/выкл (как на скрине) и кнопкой
"Изменить текст". Если текст выключен - бот просто не отправляет сообщение
на этом шаге (сделка продолжает обрабатываться).

| Текст | Когда отправляется |
|---|---|
| Не указали логин и пароль | сразу после оплаты, категория Roblox |
| Ответ после данных | сразу после того как покупатель прислал логин:пароль |
| Данные неверные | Swizzyer отклонил данные аккаунта (`credentials_rejected` и т.п.) - бот просит ввести заново |
| Данные верные | Swizzyer принял данные, отправляется ссылка для 2FA (`{url}`, `{expires}`) |
| После выполнения | заказ полностью выполнен (`status=completed`) |
| После подтверждения | покупатель подтвердил сделку на Playerok |
| Неверный формат данных | покупатель прислал текст не в формате `логин:пароль` |
| Заказ не выполнен | общая ошибка / `failed` / `cancelled` / таймаут |
| Товар не найден в каталоге | категория Roblox, но товара нет в "Товары" - ручная обработка |
| Время на 2FA истекло | `status=expired` |

В текстах "Данные верные" и "Заказ не выполнен" можно использовать
подстановки `{url}`, `{expires}`, `{reason}`.

### \U0001F4DC Логи

Последние 20 событий бота (или только ошибки). Кнопка "Обновить" подгружает
свежие записи. Каждое значимое действие бота (новая сделка, отправка
сообщения, создание заказа на Swizzyer, ошибки API, обновление статуса
сделки и т.д.) попадает в этот лог.

### \U0001F6CD Товары

Сопоставление названий товаров на Playerok с реальными
`product_id` / `sku_id` / `availability_id` / `amount` пакетов Robux
(берутся из `https://swizzyer.com/dashboard/products`). Без записи в этом
разделе сделка по такому товару уйдёт на ручную обработку (бот пришлёт
покупателю текст "Товар не найден в каталоге").

## Логика обработки сделки

1. Покупатель оплачивает товар категории Roblox -> `NEW_DEAL` / `ITEM_PAID`.
2. Бот ищет товар в каталоге ("Товары"). Если не найден - ручная обработка.
3. Бот отправляет текст "Не указали логин и пароль".
4. Покупатель присылает `почта:пароль` (или `логин|пароль`).
5. Бот отправляет текст "Ответ после данных" и создаёт заказ
   `POST /v1/orders` (`mode=hosted_link`, `credentials={username,password}`).
6. Бот опрашивает `GET /v1/orders/{id}` каждые `SWIZZYER_POLL_INTERVAL_SEC`
   секунд:
   - если `status=requires_action` с кодом, похожим на отклонённые данные
     (`credentials_rejected`, `invalid_credentials` и т.п.) - отправляет
     текст "Данные неверные" и возвращает сделку в состояние ожидания
     логина/пароля (можно вводить заново);
   - если появилась `verification.url` и данные не отклонены - отправляет
     текст "Данные верные" со ссылкой на 2FA;
   - `status=completed` - отправляет "После выполнения" и переводит сделку
     на Playerok в статус `SENT`;
   - `status=expired` - отправляет "Время на 2FA истекло";
   - `failed` / `cancelled` / `partially_delivered` - отправляет "Заказ не
     выполнен" с причиной.
7. Когда покупатель подтверждает сделку на Playerok (`DEAL_CONFIRMED`) - бот
   отправляет текст "После подтверждения".

## Замечания

- **Idempotency-Key** для создания заказа - `deal-<deal_id>`, повторное
  событие не создаст дубликат заказа на Swizzyer.
- Polling можно заменить на webhooks (`/v1/webhook_endpoints`) для меньшей
  нагрузки, но это требует отдельного HTTP-сервера с публичным URL.
- Точное соответствие кодов `failure_reason.code` для "неверные данные"
  уточняется по факту использования API - список `CREDENTIALS_BAD_HINTS` в
  `order_processor.py` можно расширить при необходимости.
